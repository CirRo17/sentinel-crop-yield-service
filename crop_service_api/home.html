Sentinel Crop Service

农作物遥感监测在线服务

提供作物分类、长势监测、病虫害检测与产量估算等遥感分析能力，通过异步 API 接口调用，支持状态轮询与结果下载。

服务地址
__APP_BASE__
当前页面、Swagger 文档和接口都挂在这个服务地址下。接口请求统一使用 /api/... 路径。
对接流程
STEP 1
上传影像与地块
调用 /api/data/upload 上传 GeoTIFF，/api/data/upload-parcels 上传地块 ZIP。
STEP 2
作物识别
调用 /api/infer/start 启动推理，下载分类图、置信度图及地块级作物分布 Shapefile。
STEP 3
产量估算
调用 /api/yield/estimate，基于分类结果按作物估算面积、单产和总产。
STEP 4
长势监测
调用 /api/growth/start，Z-Score 分析输出地块级长势等级。
STEP 5
病虫害胁迫
调用 /api/pest/start，多时相指数对比输出地块级胁迫等级。
接口能力

影像上传

接收多波段 GeoTIFF 输入，生成可追踪的文件标识，作为后续推理任务的输入。

POST /api/data/upload
可用

语义分类

按模型需要自动做缩放、归一化和指数构建，输出分类图；如传入地块数据，同步生成地块级多数类别结果。

POST /api/infer/start
可用

状态轮询

异步任务模式，查询进度、有效像元数量、整幅影像的主要类别结果和下载地址。

GET /api/infer/status/{task_id}
可用

结果下载

统一管理分类结果、置信度图、元数据 JSON；传入地块数据时可下载地块级 Shapefile ZIP。

GET /api/infer/download/*
可用

产量估算

在分类完成后，基于 NDVI 或 LAI 植被指数按作物估算逐像素产量，返回面积、单产、总产及不确定性评估。

POST /api/yield/estimate
可用

长势监测

基于多年同期 NDVI 基准的 Z-Score 距平分析，自动运行像元级分级→地块级定级，输出优/良/中/差四级评估。

POST /api/growth/start
可用

病虫害检测

多时相植被指数对比（当前 vs 前期 vs 历史同期），综合 Z-Score、趋势下降、多指数一致性评分，输出地块级胁迫等级。

POST /api/pest/start
可用
调用示例
BASH 
# 1. 上传原始 Sentinel-2 影像（文件名需含日期，如 202504、2025-04）
#    同月的多景会自动归组，服务端做对齐+中值合成
curl -X POST __API_BASE__/data/upload -F "file=@202504_S2_west.tif"
curl -X POST __API_BASE__/data/upload -F "file=@202504_S2_east.tif"
curl -X POST __API_BASE__/data/upload -F "file=@202507_S2_full.tif"
curl -X POST __API_BASE__/data/upload -F "file=@202508_S2_north.tif"
curl -X POST __API_BASE__/data/upload -F "file=@202508_S2_south.tif"

# 2. 上传地块数据（可选）
curl -X POST __API_BASE__/data/upload-parcels -F "file=@parcels.zip"

# 3. 启动推理（只需平铺 file_ids，服务端按文件名日期自动分组）
curl -X POST __API_BASE__/infer/start -H "Content-Type: application/json" \
  -d '{"file_ids": ["id1","id2","id3","id4","id5"], "parcel_file_id": "parcel-id"}'

# 4. 查询任务状态
curl __API_BASE__/infer/status/{task_id}

# 5. 下载分类结果
curl "__API_BASE__/infer/download/{task_id}?format=classification" -o classification.tif

# 5b. 下载地块级 Shapefile
curl "__API_BASE__/infer/download/{task_id}?format=shp" -o parcels.zip

# 6. 启动估产任务
curl -X POST __API_BASE__/yield/estimate \
  -H "Content-Type: application/json" \
  -d '{"task_id": "{task_id}", "index": "ndvi"}'

# 7. 查询估产状态
curl __API_BASE__/yield/status/yield_{task_id}

# 8. 下载估产结果
curl "__API_BASE__/yield/download/yield_{task_id}?format=metadata" -o yield.json

# ── 长势监测 ──
# 9. 启动长势监测（需先完成数据准备）
curl -X POST __API_BASE__/growth/start -H "Content-Type: application/json" \
  -d '{"target_year": 2026, "target_month": 6}'

# 10. 查询长势监测状态
curl __API_BASE__/growth/status/{task_id}

# 11. 下载长势结果
curl "__API_BASE__/growth/download/{task_id}?format=step3_csv" -o parcel_growth.csv

# ── 病虫害检测 ──
# 12. 启动病虫害检测（需先运行 Step1 准备历史数据）
curl -X POST __API_BASE__/pest/start -H "Content-Type: application/json" \
  -d '{}'

# 13. 查询病虫害检测状态
curl __API_BASE__/pest/status/{task_id}

# 14. 下载病虫害结果
curl "__API_BASE__/pest/download/{task_id}?format=step3_shp" -o pest_grade.zip
import requests
import time

BASE = "__API_BASE__"

# 上传原始影像（文件名含日期，如 202504_S2_west.tif）
file_ids = []
for path in ["202504_S2_west.tif", "202504_S2_east.tif",
             "202507_S2_full.tif", "202508_S2_north.tif", "202508_S2_south.tif"]:
    with open(path, "rb") as f:
        upload = requests.post(f"{BASE}/data/upload", files={"file": f}).json()
    file_ids.append(upload["file_id"])

# 可选：上传地块 Shapefile ZIP
with open("your_parcels.zip", "rb") as f:
    parcel = requests.post(f"{BASE}/data/upload-parcels", files={"file": f}).json()

# 启动推理（平铺 file_ids，服务端自动按文件名日期分组+合成）
task = requests.post(
    f"{BASE}/infer/start",
    json={"file_ids": file_ids, "parcel_file_id": parcel.get("parcel_file_id")},
).json()

while True:
    status = requests.get(
        f"{BASE}/infer/status/{task['task_id']}",
    ).json()
    print(status["status"], status["progress"], status.get("top_predictions"))
    if status["status"] in ("completed", "failed"):
        break
    time.sleep(1)

# 5. 启动估产
yield_task = requests.post(
    f"{BASE}/yield/estimate",
    json={"task_id": task["task_id"], "index": "ndvi"},
).json()

while True:
    ys = requests.get(
        f"{BASE}/yield/status/{yield_task['yield_task_id']}",
    ).json()
    print(ys["status"], ys["progress"])
    if ys["status"] in ("completed", "failed"):
        if ys.get("summary"):
            print("总产:", ys["summary"]["total_yield_kg"], "kg")
            for crop in ys.get("crops", []):
                print(f"  {crop['label']}: {crop['area_ha']:.1f} ha, {crop['mean_yield_kg_ha']:.1f} kg/ha")
        break
    time.sleep(0.5)

返回内容说明

状态接口 返回任务状态、进度、有效像元数量、模型特征名、整幅影像的主要类别结果，以及结果下载地址。

下载接口 默认提供分类图 GeoTIFF、置信度图 GeoTIFF、元数据 JSON；启动任务时传入 parcel_file_id 后额外提供地块级 Shapefile ZIP。

元数据 JSON 包含输入文件、波段映射、缩放参数、有效像元数量和输出文件路径等对接信息。
